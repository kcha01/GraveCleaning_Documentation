# PROJEKT

## DOKUMENTACJA

W dokumentacji udało się ustalić zakres projektu oraz jego cel. Podczas fazy analizy wymagań ustalono metody wydobywania wymagań oraz ich ustalenie. Zdefiniowano aktorów procesu, wymagania funkcjonalne dla poszczególnych nich i pozafunkcjonalne systemu. W dalszej części obrazowo przedstawiono diagram przypadków użycia, na którym przedstawiono aktorów i ich udział w życiu systemu. Stworzono też pięć map procesów w notacji BPMN oraz diagram klas. Do dokumentacji dołączono harmonogram oraz przykładowy projekt UI aplikacji internetowej stworzonej przy pomocy narzędzia Figma.

W celu zobaczenia dokumentacji tekstowej, proszę pobrać dokumentację i otworzyć plik w programie MS Word. Tutaj zostaną przedstawione jedynie załączniki graficzne.
